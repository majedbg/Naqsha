Problem. The same concept — “this layer guides that layer” — is implemented three times with three vocabularies (modulator/map/channel · guide/lattice/node · host/adorn/anchor), and resolution runs at four sites: the canvas resolver (first-match only), an inline lattice branch, the motif host resolver, and an independent 3D re-derivation that silently drops two channels; lattice edges never appear in the graph the rails and badges trust (modulationGraph.js:40 gates on canProduceField).

Solution. Deepen into one relationship module — a single edge model and a single resolver seam with channel adapters behind it — so rails, badges, Inspector gating, canvas/SVG, and 3D all read the same graph, and the resolver’s polymorphic return (field-shape vs nodes-shape, switched on by every consumer) is contained behind the seam.

  • leverage: one graph, five consumers
  • multi-source blending lands once, not twice
  • 3D can’t silently drop channels
  • lattice edges become visible to the UI
  • polymorphic-return leak contained
  • duplicated polarity migration deleted
Unlocks tracked gap A1 (TODO_June29): “N sources · 1 active” dead-end UI — blending needs a composite return contract; today it would have to be built twice (canvas resolver + 3D resolver).

Canvas and SVG derive from the same data — parity by construction. The plotter’s polyline model becomes the shared representation instead of a DOM-parse reconstruction.

Problem. The DrawingContext seam covers canvas-vs-headless-test but not canvas-vs-export: every pattern emits its geometry twice by hand (ctx calls + SVG strings), the real seam between live render and all export paths is the cached patternInstances’ string arrays, and three separate parsers re-derive structure from the app’s own strings.

Solution. Make structured geometry the primary artifact a pattern produces and push SVG serialization to the edge — staged: start with the stamping family (motif · extracted · imported already share the shape; MotifPattern’s per-instance-matrix dual-emit is the working model), then migrate string-based patterns behind the existing golden-master snapshots.

  • canvas == SVG by construction
  • delete three re-parser implementations
  • symmetry: one implementation, not two
  • variable weight becomes canvas-visible
  • plotter loses its DOMParser dependency
  • 18 SVG snapshots guard the migration
Biggest blast radius in this review (~22 pattern files + 5 consumers). The badge is amber for cost, not for doubt — the divergences are real and documented in-repo (fabricationDivergence.test.js pins the old three-way disagreement).

Problem. There is no canonical document schema — layers are hand-built at four sites with divergent fields, six persistence surfaces serialize their own subsets under three unrelated version fields, document-level migration is dead code, and persisted operations/machineProfile are written but never read back.

Solution. Promote the discipline that already exists in documentSnapshot.js (single enumeration point, restore(capture()) === noop invariant) into a document module owning the schema, the layer factory, and migration — every persistence surface becomes an adapter over its capture/restore interface.

  • the schema becomes code, not a comment
  • one migration seam, one version
  • layer fields guaranteed, not back-filled
  • write-only fields become honest (read or drop)
  • adapters already exist — real seam today
  • undo checksum LIMITATION resolves
